# 仓库功能与端到端工作流

本文档对齐当前目录结构（尤其是 `outputs/` 与 `tools/*` 分组）并按推荐执行顺序给出命令。

## 0. 当前结构重点

- 代码：`pp_nextgen/`
- 配置：`configs/`
- 产物：`outputs/profiling`、`outputs/scheduler`、`outputs/runtime`
- 工具：`tools/profiling`、`tools/scheduler`、`tools/runtime`、`tools/simulation`、`tools/transport`

## 1) Capture（采集 profile）

入口脚本：

- `tools/profiling/capture_split_module_profiles.py`

示例：

```powershell
python tools/profiling/capture_split_module_profiles.py --cuda-device 0 --device-id 3060 --output-dir ./my_profiles
```

## 2) Build registry（构建设备注册表）

入口脚本：

- `tools/profiling/build_registry.py`

示例：

```powershell
python tools/profiling/build_registry.py `
  --inputs ./my_profiles `
  --model-config configs/model/llama2_7b.yaml `
  --out outputs/profiling/registry/device_registry.v3.json `
  --emit-all-results outputs/profiling/all_results
```

## 3) Schedule strategy（策略求解）

入口脚本：

- `tools/scheduler/solve_strategy.py`

示例：

```powershell
python tools/scheduler/solve_strategy.py --cluster configs/cluster/all_devices.yaml
```

默认输出：

- `outputs/scheduler/export/pipeline_strategy.json`
- `outputs/scheduler/export/tail_sweep_report.json`
- `outputs/scheduler/export/workers/*.strategy.json`

## 4) Runtime / Simulator（运行时与仿真）

运行时相关：

- `tools/runtime/submit_task_to_master.py`
- `tools/runtime/gen_runtime_proto.py`

仿真相关：

- `tools/simulation/run_pipeline_des_sim.py`
- `tools/simulation/run_cluster_sim.py`

DES 示例：

```powershell
python tools/simulation/run_pipeline_des_sim.py `
  --strategy outputs/scheduler/export/pipeline_strategy.json `
  --arrival-rate 6 `
  --duration-s 10 `
  --batch-size 1 `
  --context-len 128 `
  --target-len 256 `
  --out outputs/runtime/simulation/pipeline_des.sim_metrics.json `
  --master-latency-sidecar-out outputs/runtime/simulation/pipeline_des.master_latency.json
```

本机 gRPC 联通模拟示例：

```powershell
python tools/simulation/run_cluster_sim.py `
  --pipeline-strategy outputs/scheduler/export/pipeline_strategy.json `
  --model-config configs/model/llama2_7b.yaml `
  --workers-export-dir outputs/scheduler/export/workers `
  --runtime-logs-dir outputs/runtime/logs
```

## 命名一致性要求

1. `device`（capture 输出）= `device_type`（registry）= `device_group` 键（cluster）。
2. `designated_device` 是设备类型，不是 worker 名。
3. worker 名必须与 `pipeline_strategy.json`、`workers/*.strategy.json` 完全一致。
